
# ADR-001: Adopt Spring Boot as Application Platform

**Status:** Accepted
**Date:** 2026-02-17

---

## Context

Проект использовал устаревший Spring / DAO-подход с ручной конфигурацией и слабой стандартизацией структуры.

Это усложняло:

* поддержку
* тестирование
* архитектурную эволюцию

Требовалась платформа, обеспечивающая:

* единый способ конфигурации
* стандартную структуру backend-приложения
* удобную интеграцию с persistence-слоем
* поддержку тестирования

---

## Decision

Принято решение использовать Spring Boot 3.x как базовую платформу приложения.

Используемые компоненты:

* Spring Web (REST)
* Spring Data JPA
* Spring Boot Test

Spring Boot используется как application platform, определяющая:

* модель конфигурации
* структуру приложения
* жизненный цикл запуска
* подход к тестированию

---

## Alternatives Considered

1. Классический Spring без Boot
   → отклонён из-за конфигурационной сложности.

2. Другие фреймворки (Quarkus, Micronaut)
   → отклонены на текущем этапе для сохранения максимальной стандартности стека.

---

## Consequences

### Positive

* Снижение boilerplate-кода
* Стандартизированная структура
* Упрощённая конфигурация
* Улучшенная интеграция с тестированием

### Negative

* Привязка к Spring ecosystem
* Более высокий memory footprint по сравнению с lightweight frameworks

---

## Architectural Impact

Это решение стало основой для:

* перехода к слоистой архитектуре
* выделения domain-слоя
* внедрения идемпотентности
* интеграционных тестов с Testcontainers
